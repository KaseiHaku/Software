使虚拟机桌面大小，随虚拟机窗口大小自动调整：
    View -> Autosize -> 选择 Autofit Guest

从 Host SSH 连接到虚拟机：
    1. 打开 windows 防火墙 ping 配置
       Control Panel -> System and Security -> Windows Defender Firewall -> Advanced Setting -> Inbound Rules 
       -> File and Printer Sharing(Echo Request - ICMPv4-in)  Enable
       -> File and Printer Sharing(Echo Request - ICMPv6-in)  Enable

如果 window 更新导致，虚拟机任何问题
    admin 权限重新执行 安装包，选择 repair 修复即可

VMX 进程 CPU 100%：
    可能跟 Hyper-V 冲突，建议关闭该 window feature
    Win+R -> services.msc -> HV Host Service: disable
    Win+R(Admin) -> cmd -> bcdedit /set hypervisorlaunchtype off

创建 Windows 共享目录
    关闭虚拟机 -> 右键 -> Settings -> Options -> Shared Folders
        Always enabled
            Add...
                Host path: E:\
                Name: WinShareDir                    # 这里名称必须和下面命令保持一致
    保存
    打开虚拟机 -> 打开 Terminal 
        shell> vmware-hgfsclient                    # 如果回显 WinShareDir 则表示开启成功
        shell> mkdir /home/kasei/Desktop/WinShareDir        # 如果该目录不存在，则先创建
        # 执行挂载命令，重启失效，可以写自动脚本；需要放出 /etc/fuse.conf 中的 user_allow_other 注释
        shell> vmhgfs-fuse .host:/WinShareDir /home/kasei/Desktop/WinShareDir -o allow_other -o uid=$(id -u) -o gid=$(id -g)    
        shell> umount /home/kasei/Desktop/WinShareDir              # 卸载共享目录

        # 复制本机项目到 windows 中
        shell> \cp -a --update=all -f /home/kasei/Desktop/GitLocalRepository/kh-app-client/app-yjyuan/dist/dev/mp-weixin/ /home/kasei/Desktop/WinShareDir/GitLocalRepository/app-yjyuan
        # 对于需要排除指定目录的情况
        
        shell> cd /project/src && find . -type f \
                ! -ipath '*.git*' -prune  \
                ! -ipath '*.idea*' -prune \

        shell> rsync -av --delete --exclude='*/node_modules/*' --exclude='.git' --exclude='*.log' source/ destination/        # 需要装 rsync
        shell> rsync -av --delete --exclude={'*/node_modules/*','.git','*.log'} source/ destination/                          # 需要装 rsync

当 VPN 软件只支持 windows 时， 虚拟如何使用 VPN：
    - 选中对应虚拟机 > 右键  > Settings...
    - Hardware 选项卡 > Network Adapter > Network connection > [选中] NAT: Used to share to host's IP address

    #### 如果需要通过其他网络访问虚拟机中的服务，怎么办????? 配置 Port Forward 即可，操作如下
    - 点击 VMware Workstation 中的 Edit 菜单 > Virtual Network Editor... > Change Settings 
    - 选择 Name: VMnet8   Type: NAT 类型的条目
    - 修改以下值 Subnet IP: 192.168.23.0    Subnet mask: 255.255.255.0
    - 点击 DHCP Settings...
        Starting IP address: 192.168.23.128
        Ending IP address: 192.168.23.254
    - 点击 NAT Settings... 
    - Gateway IP: 192.168.23.2(需要跟上面 Subnet ip/mask 在同一网段)
    - Port Forwarding 中，点击 Add... 按钮添加如下条目
    - Host port: 50080   Type: TCP    Virtual machine IP address: 192.168.23.106(需要跟上面 Subnet ip/mask 在同一网段) Virtual machine port: 50080
      Host port: 50443   Type: TCP    Virtual machine IP address: 192.168.23.106(需要跟上面 Subnet ip/mask 在同一网段) Virtual machine port: 50443   
    - 点击 OK 完成配置
    - 启动虚拟机 > 打开网卡配置
      Method: Manual（静态 IP）
      DNS Servers: 8.8.8.8,223.5.5.5
      Address: 192.168.23.106(需要跟上面 Subnet ip/mask 在同一网段，并且和 Port Forwarding 中的相同)
      Netmask： 255.255.255.0
      Gateway: 192.168.23.2(需要跟上面 Gateway IP 中的相同)
    - 保存即可
    - @trap 注意外部访问地址为 宿主机地址，所以微服务注册时的注册地址要改成 宿主机 的地址，不能是 虚拟机中的 IP(即：192.168.23.106 )
        

################################ VMWare ESXi ################################
从 ESXi 导出虚拟机：
    下载并安装 VMware-ovftool： 
        https://my.vmware.com/group/vmware/downloads/details?downloadGroup=OVFTOOL440&productId=974

    在默认目录（C:\Program Files\VMware\VMware OVF Tool）下运行CMD窗口
        
    登陆 ESXi 查看虚拟机信息
        cmd> ovftool.exe vi://192.168.1.50/k8s-worker3
    使用ovftool将虚拟机导出为OVA格式
        cmd> 创建一个空目录
        cmd> ovftool.exe vi://192.168.1.50/centos7-template E:\k8s
